## Описание проекта

Проект состоит из двух Docker-контейнеров:

1. **Generator** — генерирует CSV-файл с данными о товарах.
2. **Reporter** — анализирует CSV-файл и создаёт HTML-отчёт.

Данные сохраняются в локальную папку `data`, которая монтируется внутрь контейнеров как `/data`.

## Команды

Сборка генератора:

```bash
./run.sh build_generator
```

Запуск генератора:

```bash
./run.sh run_generator
```

Создание данных локально без Docker:

```bash
./run.sh create_local_data
```

Сборка аналитика:

```bash
./run.sh build_reporter
```

Запуск аналитика:

```bash
./run.sh run_reporter
```

Показать структуру проекта:

```bash
./run.sh structure
```

Удалить все сгенерированные данные:

```bash
./run.sh clear_data
```

Проверить содержимое папки `/data` внутри контейнера генератора:

```bash
./run.sh inside_generator
```

Проверить содержимое папки `/data` внутри контейнера аналитика:

```bash
./run.sh inside_reporter
```

Запустить веб-сервер с отчётом:

```bash
./run.sh report_server
```

## Проверка работы проекта

Создать данные:

```bash
./run.sh build_generator
./run.sh run_generator
```

Создать отчёт:

```bash
./run.sh build_reporter
./run.sh run_reporter
```

После выполнения в папке `data` должны появиться файлы:

```text
data.csv
report.html
```

## Запуск отчёта через GitHub Codespaces

Запустить веб-сервер:

```bash
./run.sh report_server
```
Если открылась пустая страница или ошибка, нужно добавить в конец ссылки 
```text
/report.html
```

После запуска:

1. Открыть вкладку **Ports**.
2. Найти порт **8080**.
3. Нажать на ссылку напротив порта.
4. Откроется HTML-отчёт.

## Тематика данных

В проекте генерируются данные о товарах магазина:

* price — цена товара;
* rating — рейтинг;
* stock — количество на складе;
* category — категория товара.

Отчёт автоматически вычисляет статистику по числовым колонкам и показывает распределение значений для категориальных данных.
